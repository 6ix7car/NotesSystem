# 📝 Система заметок — инструкция пользователя

## 🔐 Аутентификация

| Команда | Описание | Пример |
|---------|----------|--------|
| `--login <username> <password>` | Вход в систему | `--login admin mypass` |
| `--register <username> <password> [role]` | Регистрация нового пользователя. Роли: `admin`, `user`, `readonly` (по умолчанию `user`) | `--register john 12345 user` |
| `--myrole` | Показать роль текущего пользователя | `--myrole` |
| `--logout` | Выход из системы | `--logout` |

## 📝 Работа с заметками

> **Примечание по ролям:**  
> - `admin` и `user` — могут создавать, редактировать, удалять и восстанавливать заметки.  
> - `readonly` — могут только просматривать заметки (`--listNotes`, `--getNote`).

| Команда | Описание | Пример |
|---------|----------|--------|
| `--addNewNote "<текст>"` | Создать новую заметку | `--addNewNote "Проверить сервер БД"` |
| `--listNotes` | Показать все свои заметки | `--listNotes` |
| `--getNote <id>` | Показать заметку по ID | `--getNote 5` |
| `--editNote <id> "<новый текст>"` | Редактировать заметку | `--editNote 5 "Новый текст заметки"` |
| `--deleteNote <id>` | Удалить заметку (мягкое удаление) | `--deleteNote 5` |
| `--restoreNote <id>` | Восстановить удалённую заметку | `--restoreNote 5` |

## 📊 Мониторинг системы (только для роли `admin`)

| Команда | Описание | Пример |
|---------|----------|--------|
| `--systemStats local` | Показать статистику текущего сервера (CPU, RAM, HDD) | `--systemStats local` |
| `--systemStats remote <host> <user> <pass>` | Показать статистику удалённого сервера (требуется SSH/WMI) | `--systemStats remote 192.168.1.100 root pass` |

## 📋 Логи безопасности (только для роли `admin`)

| Команда | Описание | Пример |
|---------|----------|--------|
| `--securityLogs list` | Показать последние 50 записей логов безопасности | `--securityLogs list` |
| `--securityLogs export json` | Экспортировать логи в JSON | `--securityLogs export json` |
| `--securityLogs export csv` | Экспортировать логи в CSV | `--securityLogs export csv` |

## 🔄 Обновления

| Команда | Описание | Пример |
|---------|----------|--------|
| `--checkUpdate` | Проверить наличие обновлений | `--checkUpdate` |
| `--update` | Выполнить обновление (если доступно) | `--update` |

## ❓ Справка

| Команда | Описание | Пример |
|---------|----------|--------|
| `--help` | Показать краткую справку в консоли | `--help` |
| `exit` | Выйти из программы | `exit` |

## Примечания

- Пароли хешируются с помощью **MD5**.
- При трёх неудачных попытках входа аккаунт блокируется на 15 минут.
- Заметки при удалении не стираются из базы, а помечаются как удалённые (`IsDeleted = true`), что позволяет их восстановить.
- Логи безопасности сохраняются как в базу данных, так и в текстовый файл `logs\security.log`.
- Для работы с PostgreSQL требуется запущенный сервер и созданная база данных `NotesSystemDB`.
